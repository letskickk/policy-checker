# AI 공약 멘토링 시스템 — 요구사항 명세서

## I. 추진배경

| 구분 | 내용 |
|------|------|
| **문제** | 선거 준비 과정에서 출마자는 공약이 당 정강·정책기조에 부합하는지, 타 후보 공약과의 중복·유사 여부를 수시로 점검해야 함 |
| **제약** | 정책국의 인력·시간 제약으로 상시 지원이 어려워 반복 문의가 누적되는 구조 |
| **촉발** | 정책위원회–시·도당위원장 연석회의에서 "당 기본방향 가이드가 없어 지역공약을 매칭하기 어렵다"는 의견 다수 제기 |
| **방향** | 출마자가 **언제든 기준에 맞춰 스스로 점검**할 수 있는 AI 공약 멘토링 시스템 구축 |

---

## II. 추진목적

1. **출마자 경험**
   - 공약을 입력하면 시스템이 **즉시** 다음을 제공:
     - (1) **당 방향 부합 여부**
     - (2) **중복·유사 여부**
     - (3) **보완점**
   - 즉, **AI 공약 멘토링** 환경 구축

2. **정책국 역할**
   - 기초 검토를 **자동화**
   - 전략·메시지·공약 완성도에 **집중**

---

## III. 추진범위

- **본 시스템**: 즉각 피드백 + **당 정체성 가이드**에 집중
- **역할 분리**: 참치상사에서 개발 중인 **지방의회 회의록 기반 지역 현안 발굴/데이터 분석**과는 별도
  - 본 시스템: 당 방향·중복·보완점 (공약 품질·일관성)
  - 참치상사: 지역 현안 발굴·데이터 분석 (지역 이슈·자료)

---

## IV. 핵심 기능

### 1. 당 방향 부합 점검

| 항목 | 설명 |
|------|------|
| **기준** | 정강·정책·과거 공약 |
| **산출** | 부합 여부 점검 |
| **안내** | 수정·보완이 필요한 항목을 **체크리스트**로 제공 |

**세부 요구사항(예시)**
- 입력: 출마자 공약 문장/목록
- 출력: 부합/미부합/부분부합 + 해당 정강·정책·과거 공약 근거, 수정 제안

### 2. 중복·유사 탐지

| 항목 | 설명 |
|------|------|
| **기준** | 후보 공약 DB |
| **산출** | 유사 공약 제시 |
| **안내** | **차별화 포인트** 제안 |

**세부 요구사항(예시)**
- 입력: 출마자 공약
- 출력: 유사한 타 후보 공약 목록, 유사도, 차별화 방향 제안

## V. 기능 요약표

| 기능 | 입력 | 기준 데이터 | 출력 |
|------|------|-------------|------|
| 당 방향 부합 점검 | 공약 텍스트 | 정강·정책·과거 공약 | 부합 여부 + 체크리스트(수정·보완 항목) |
| 중복·유사 탐지 | 공약 텍스트 | 후보 공약 DB | 유사 공약 목록 + 차별화 포인트 제안 |

---

## VI. 비기능 요구사항(권장)

- **응답 속도**: 공약 입력 후 **즉시** 피드백 (목표: 수 초 이내)
- **가용성**: 출마자가 언제든 접근 가능한 환경 (웹/챗 등)
- **보안**: 공약·후보 정보 접근 권한 및 저장 정책 정의
- **확장성**: 정강·정책·공약 DB 추가·갱신 시 유지보수 용이

---

## VII. 확정·추후 검토 항목

**확정**
- 정강·정책·과거 공약 원본: **PDF** 방식 제공
- 분석: **GPT API** 이용
- 1차 범위: **당 부합 점검** 우선

**추후 검토**
- **후보 공약 DB** 구축·갱신 주기 및 관리 주체 (중복·유사 탐지용)
- UI 형태 (웹 폼, 챗봇, API 등)
- 정책국 검토 워크플로와의 연동 (검토 결과 반영 여부 등)
