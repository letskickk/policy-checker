# 지역별 출마자 공약 등록 방법

지도 페이지([/map](https://policy.reformparty.kr/map))에 노출되는 **출마자**와 **공약**을 등록·수정하는 방법입니다.

## 데이터 구조

- **출마자(candidates)**: 후보명, 지역(시/도·선거구), 선거 유형(단체장/광역의원/기초의원 등)
- **공약(candidate_pledges)**: 출마자별 공약 제목·카테고리·우선순위 (한 명당 여러 개)

지도에서 지역을 선택하면 해당 지역의 출마자 목록과 각 후보의 핵심 공약(최대 3개)이 표시됩니다.

## 등록 방법

### 1) 관리자 페이지에서 등록 (권장)

1. **관리자 로그인** 후 **[관리자](https://policy.reformparty.kr/admin)** 메뉴 접속
2. **출마자·공약 등록** 링크로 이동
3. **시/도 → 선거구 → 선거 유형** 선택 후, **후보명**과 **공약**을 입력해 저장

한 번 저장하면 지도 페이지에서 해당 지역을 선택했을 때 바로 노출됩니다.

### 2) API로 직접 등록

관리자 권한(로그인 쿠키)이 있는 상태에서 다음 API를 호출합니다.

**후보 등록 (POST)**

```
POST /api/admin/candidates
Content-Type: application/json
```

| 필드 | 필수 | 설명 |
|------|------|------|
| name | O | 후보명 |
| region_code | O | 시/도 코드 (예: 11=서울, 41=경기) |
| district_name | | 선거구명 (예: 마포구, 마포구 가선거구) |
| district_code | | 선거구 코드 (예: 11:마포구, 11:마포구:가선거구) |
| election_type | | local_council, regional_council, local_mayor, metro_mayor, education 등 |
| election_level | | regional 등 |
| pledges | | 공약 배열: [{ "title": "공약 제목", "category": "복지", "priority": 1 }, ...] |

- **수정**: `PUT /api/admin/candidates/{candidate_id}` (동일 body)
- **목록 조회(공개)**: `GET /api/candidates?region_code=11&district_code=11:마포구&election_type=local_council`

## 선거 유형(election_type) 값

| 값 | 설명 |
|----|------|
| metro_mayor | 광역단체장 (시도지사) |
| regional_council | 광역의원 (시도의원) |
| local_mayor | 기초단체장 (구청장 등) |
| local_council | 기초의원 (구의원 등) |
| education | 교육감 |

## 지도 페이지 동작

- **지역 클릭** → 해당 시/도의 출마자 수가 배지로 표시되고, 선거 유형·선거구 선택 후 출마자 목록 표시
- **출마자 데이터가 없으면** "해당 지역의 출마자 정보가 없습니다" 메시지 표시
- 데이터는 `candidates`·`candidate_pledges` 테이블에서 조회되며, 관리자 등록 또는 API로만 추가됩니다.
